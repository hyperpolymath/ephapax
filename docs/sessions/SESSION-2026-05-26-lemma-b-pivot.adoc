== The Option 2 obstacle

Option 2 in PRESERVATION-HANDOFF.md is "Inversion on `Hstep` with
structural recursion ... `~150 LOC, orthogonal to the current case
split`". On closer inspection it has a real obstacle the doc did not
anticipate.

=== The counterexample

[source,coq]
----
e  = ELet (ERegion r v_inner) e2     (* well-typed at R with In r R *)
e' = ELet v_inner e2                  (* after S_Let_Step / S_Region_Exit *)
R' = remove_first r R                 (* r exited from inside *)
----

`e2` may syntactically reference `r` (because `T_Let` at
`formal/Typing.v:103` has no `~In r (free_regions T)` or
`expr_free_of_region` premise on siblings). After the step, `R'` has
no `r`, so `e2`'s `r`-references prevent retyping `e'` at `R'`.

The handoff doc anticipated this for the `expr_free_of_region`
extraction path: "a `step_exit_implies_free_of_exited_region` lemma
would be FALSE for congruence cases that preserve siblings — the
sibling's surviving `r`-references break the freedom claim". The
doc's Option 2 was framed as **orthogonal** to the freedom-extraction
path; on closer inspection the same counterexample blocks Option 2's
congruence cases — they need to *rebuild* `e'`'s typing at `R'`, and
the rebuild requires `e2` to typecheck at `R'`, which `e2`'s
`r`-references prevent.

=== Verification chain

. `T_Let` in `formal/Typing.v` lines 103-106 has no sibling-freedom
  premise.
. `S_Region_Exit` in `formal/Semantics.v` lines 216-221 has
  `expr_free_of_region r v` only on the value being lifted out — not
  on siblings.
. `S_Let_Step` likewise has no freedom-on-siblings premise.
. ∴ `ELet (ERegion r v_inner) e2` with `e2` referencing `r` is
  syntactically and semantically constructible within the language as
  defined.
