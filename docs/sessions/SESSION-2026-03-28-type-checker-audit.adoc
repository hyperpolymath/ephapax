== Formal Proofs (Rocq 9.1.1 — 20 Qed, 6 Admitted, 0 errors)

=== Key Breakthrough: `flags_only_increase` Qed

Previously blocked by Rocq 9.1.1 `congruence`/`discriminate`/`injection`
failing on `option (ty * bool)` in inductive proofs. Solved by adding
projected accessors:

* `ctx_lookup_ty : ctx -> var -> option ty`
* `ctx_lookup_flag : ctx -> var -> option bool`
* `ctx_lookup_cons_zero_flag_contra` — proves `Some(T,true) = Some(T0,false) -> False`

These avoid `option (ty * bool)` discrimination entirely.

=== Root Cause: String.length / List.length Shadowing

`Coq.Strings.String.length` shadows `Coq.Lists.List.length` across module
boundaries. Typing.v exports lemmas using `List.length`, but Semantics.v
(which imports String) resolves `length` to `String.length`, causing
`eapply`/`eassumption` to fail on type mismatches.

Fix: import `Coq.Strings.String` BEFORE `Coq.Lists.List` so `List.length` wins.
