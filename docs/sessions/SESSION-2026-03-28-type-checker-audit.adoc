== Rocq 9.1.1 Workaround: Projected Lookups

`congruence`/`discriminate`/`injection` fail on `option (ty * bool)` inside
complex inductive proofs. **Solved** by adding projected accessors:

* `ctx_lookup_ty : ctx -> var -> option ty`
* `ctx_lookup_flag : ctx -> var -> option bool`
* `ctx_lookup_cons_zero_flag_contra` — proves `Some(T,true) = Some(T0,false) -> False`
  via functional extraction + `discriminate` on `option bool`

This unblocked `flags_only_increase` (now **Qed**).

== Remaining Theoretical Gap

`typing_ctx_transfer` T_Let/T_LetLin/T_Lam/T_Case cases:

The simplified 3-conjunct transfer (typing + types_agree + false_preserved)
doesn't guarantee that consumed flags are exactly `true` in the transferred
output. T_Let requires `(T1, true) :: G_tail` but transfer may produce
`(T1, u') :: G_tail` where `u'` could be `false` if the original consumption
wasn't tracked.

Fix: add a 4th conjunct: "consumed positions (false->true in original) are
also consumed (true) in transferred output." This was the original formulation
but was dropped because consumption chaining was hard. With the projected
lookup infrastructure now in place, it may be feasible.
