== Verdict

*Current CRG grade: **D (Alpha), close to C but not over the bar**.*

Self-declared grade: *C*. Audited grade: *D*. Overclaim of one grade.

This is a *real* project — the core-proof posture alone (0 Admitted Coq,
0 `believe_me` Idris2, dual-formalisation end-to-end) is rare in the
estate and would put ephapax ahead of most C-rated repos on the proof
dimension. The gap to C is *not* about the language — it is about three
operational items: CI green, per-crate README coverage, and a dated
dogfooding log. All three are tractable.

C is *not* earned today. What's missing for C, concretely:

1. *CI green*. `rust-ci.yml` has been failing at workflow-parse on every
   push for at least three days running. Fix the workflow file (or the
   `status-gate.sh` it calls) so the test, fmt, clippy, and Coq-proof
   jobs actually run to green. Without this, the "357 tests pass" claim
   has no continuous attestation.
2. *Per-crate README coverage*. 1/19 → target >=17/19 with short
   purpose-and-entry-points READMEs. The crates where code is densest
   (`ephapax-typing`, `ephapax-wasm`, `ephapax-parser`, `ephapax-desugar`)
   are the most urgent.
3. *`[dogfooding-status]` section in STATE.a2ml* with dated entries for
   the conformance-suite dogfood, the `ephapax-linear` sibling, and any
   downstream consumer (even internal prototypes). `docs/practice/` does
   not exist yet — create it with a `DOGFOOD-LOG.adoc`.

Three smaller but C-adjacent fixes:

4. *Reconcile the "Admitted" drift.* Rewrite `ROADMAP.adoc` and
   `PROOF-NEEDS.md` to reflect *0 Admitted*, mirroring STATE.a2ml and the
   live `.v` files. Also reconcile the Qed count (STATE says 67, live
   says 64).
5. *Bring `6a2/` to the full A2ML set.* Add `AGENTIC.a2ml`,
   `NEUROSYM.a2ml`, `PLAYBOOK.a2ml`. (`ephapax-proven-sys/` nested repo
   already has AGENTIC + NEUROSYM as templates to copy from.)
6. *Fill the workflow gap.* Add `hypatia-scan.yml` (critical per CLAUDE.md)
   and `quality.yml` to clear the 15→17+ gap. Consider sweeping the OCaml
   scratch files (`ast.ml`, `lexer.mll`, `parser.mly`, `typecheck.ml`,
   plus the `test_minimal.*` artefacts) into `_exploratory/` per the
   Me-scaffolding precedent.

What blocks promotion to B (after C):

1. Zero external targets. Need 6+ diverse ones. The ECOSYSTEM.a2ml
   already names five neighbours (`typed-wasm`, `typell`, `affinescript`,
   `katagoria`, `nextgen-typing`); *using* one or more of those as a
   consumer would count as a dogfood target, not an external target —
   need genuine third-party usage.
2. No `[issues-fed-back]` pipeline. Need a closed-issue trail with
   external-reporter labels.
3. The LaTeX drafts (`arcvix-*.tex`) are publication-adjacent, not
   external-user signals.
